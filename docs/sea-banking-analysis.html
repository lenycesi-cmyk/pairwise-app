2. Providers disponibles

Brick (onebrick.io) — Recommandé #1

  • Couverture : Indonésie (90%+ des banques), Singapour, Philippines
  • Tarif : ~$0.54/mois par compte connecté
  • Sandbox gratuit disponible
  • Le plus proche d'un "Plaid pour SEA"

Finverse — Meilleure couverture géographique

  • Couverture : Singapour, Hong Kong, Thaïlande, Indonésie, Philippines, Vietnam, Inde
  • API REST similaire à Plaid — intégration facilitée
  • Tarif non public (contacter sales)

Brankas (brankas.com)

  • Couverture : Indonésie, Philippines, Thaïlande (beta)
  • Tarif : $0.10/paiement, $0.50/relevé bancaire
  • Plutôt orienté paiements + data
Recommandation actuelle : Ne pas implémenter de provider SEA maintenant — Brick et Finverse sont moins matures que Plaid. Priorité aux fonctionnalités core. Quand l'expansion SEA sera envisagée, Finverse est probablement le meilleur choix (couverture la plus large, API REST compatible avec l'architecture existante).

Risque principal : Si l'app est distribuée publiquement en Indonésie ou aux Philippines, les régulateurs pourraient demander une déclaration d'activité FinTech même pour du read-only. À anticiper si le projet dépasse l'usage privé couple.
